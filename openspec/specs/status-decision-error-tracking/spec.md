## Purpose

定义变更级状态文件（.status.md）的关键决策记录表和结构化错误日志表扩展，提升变更过程的可追溯性。
## Requirements
### Requirement: 关键决策记录表

变更级 `.status.md` 文件 SHALL 包含「关键决策记录」表，用于追溯变更过程中的关键技术决策。

#### Scenario: 决策记录表结构
- **WHEN** 变更级 .status.md 被创建或更新
- **THEN** 文件 SHALL 在「审查记录」章节后包含「关键决策记录」章节
- **AND** 表格包含以下列：序号、决策点、候选方案、选择、理由、决策时间、决策阶段

#### Scenario: 决策记录初始状态
- **WHEN** 变更刚创建且尚无决策记录
- **THEN** 表格为单行 `| - | - | - | - | - | - | - |` 占位
- **AND** 有决策产生时替换为实际内容

#### Scenario: 决策记录追加
- **WHEN** 任何阶段产生了影响后续阶段的技术决策
- **THEN** 系统 SHALL 在决策表中追加一行
- **AND** 序号自增，决策时间使用 YYYY-MM-DD HH:MM 格式
- **AND** 候选方案列列出所有被考虑的方案（用 `/` 分隔）

### Requirement: 结构化错误日志表

变更级 `.status.md` 文件 SHALL 包含「错误日志」表，替代原有的简单「阻碍记录」表。

#### Scenario: 错误日志表结构
- **WHEN** 变更级 .status.md 被创建或更新
- **THEN** 文件 SHALL 在「执行记录」章节前包含「错误日志」章节
- **AND** 表格包含以下列：序号、时间、阶段、子变更、错误描述、尝试次数、解决方案、状态

#### Scenario: 错误日志初始状态
- **WHEN** 变更刚创建且尚无错误
- **THEN** 表格为单行 `| - | - | - | - | - | - | - | - |` 占位

#### Scenario: 错误记录追加
- **WHEN** 任何阶段出现需记录的错误或阻碍
- **THEN** 系统 SHALL 在错误日志表中追加一行
- **AND** 「尝试次数」列跟踪该错误的修复尝试轮数
- **AND** 「状态」列使用 ✅ 已解决 / 🔄 处理中 / ❌ 已升级 三种状态值

#### Scenario: 与子变更错误关联
- **WHEN** 错误发生在特定子变更
- **THEN** 「子变更」列填写对应的子变更名（如 `user-auth`）
- **AND** 变更级全局错误（如环境问题）填写 `-` 或 `全局`
