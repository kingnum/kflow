## MODIFIED Requirements

### Requirement: 关键词意图优先级

系统 SHALL 为意图识别关键词映射定义优先级，当多个关键词同时命中时按优先级裁决。

> **变更说明**：新增 triage 路由关键词组（反馈/报告问题/报bug/提bug → kflow-bug-triage），与修复类关键词同级（优先级 1）。修改"修复"关键词增加上下文判断逻辑：有活跃 bug-fix 子阶段时路由到 bug-fix，否则路由到 triage。原有"问题"关键词从修复类移至 triage 类。

#### Scenario: 优先级排序
- **WHEN** 系统进行意图识别
- **THEN** 关键词优先级从高到低为：
  1. 问题诊断类（反馈、报告问题、报bug、提bug、问题）→ kflow-bug-triage
  2. 修复类（修复、Bug、缺陷）→ 上下文判断（见下方）
  3. 测试类（测试、QA、验收、E2E）
  4. 归档类（归档、完成、结束）
  5. 流程类（流程、指引、帮助、如何）
  6. 创建类（新功能、开始、创建、开发）

#### Scenario: 修复类关键词上下文判断
- **WHEN** 用户输入命中修复类关键词（修复、Bug、缺陷）
- **THEN** 系统 SHALL 检查当前活跃变更是否存在正在执行的 bug-fix 子阶段
- **AND** 若存在 → 路由到 kflow-bug-fix（继续已有修复工作）
- **AND** 若不存在 → 路由到 kflow-bug-triage（新问题诊断）
- **AND** 若无法判断 → 使用 AskUserQuestion 询问用户意图

#### Scenario: 多关键词同时命中
- **WHEN** 用户输入同时命中多个优先级的关键词（如"测试一下新功能"命中优先级3"测试"和优先级6"新功能"）
- **THEN** 系统按最高优先级裁决
- **AND** 路由到优先级最高的匹配 Skill
- **AND** 在指引输出中注明检测到的多意图及裁决依据

#### Scenario: 相同优先级多关键词命中
- **WHEN** 用户输入命中同一优先级内的多个关键词
- **THEN** 系统列出匹配项供用户选择
- **AND** 不使用 AskUserQuestion 阻塞，而是在指引输出中提供可点击选项

#### Scenario: 无关键词命中
- **WHEN** 用户输入无法匹配任何关键词
- **THEN** 系统显示全套流程概览
- **AND** 引导用户选择阶段
