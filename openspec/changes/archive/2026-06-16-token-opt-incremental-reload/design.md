## Context

子代理每次冷启动必须完整重读所有 RELOAD 文件。对于大型变更，RELOAD 开销显著。

## Goals / Non-Goals

**Goals:**
- 子代理收到"已验证文件"标记后可跳过完整读取
- 保持文件变化检测的准确性

**Non-Goals:**
- 不改变 RELOAD 清单内容
- 不改变 mtime 检测机制

## Decisions

### D1: 已验证标记格式

主 Agent 在子代理 prompt 中注入：
```markdown
## RELOAD 已验证文件 (mtime 未变, 本会话已读取)
以下文件自上次读取后未发生变化，你可以信任主 Agent 提供的摘要信息：
- CONTEXT.md (术语表): {2行摘要}
- functional-designs/index.md: {1行摘要}
- detailed-design.md: {3行摘要, 仅当前子变更相关域}
```

### D2: 子代理行为

收到"已验证"标记的子代理：
- 不重新完整读取标记文件
- 如需引用标记文件的具体内容，使用主 Agent 提供的摘要
- 如果子代理在執行中发现需要标记文件的完整内容（超出摘要范围），自行读取

### D3: 安全保障

- 已验证标记仅在当前子代理调用内有效
- 下一个子代理调用时重新检测 mtime
- 文件 mtime 发生变化时不生成已验证标记

## Risks / Trade-offs

| 风险 | 缓解 |
|------|------|
| 摘要信息不足导致子代理误判 | 子代理保留自行读取的权利；摘要包含关键信息提示 |
| mtime 精度问题 | 沿用现有 mtime 检测规则 |
