## Context

重复制模式下 10 轮子代理完全独立，后期轮次重复分析已确认无问题的区域。

## Goals / Non-Goals

**Goals:**
- 后期轮次子代理接收前轮摘要，跳过已确认区域
- 保留全量遍历的兜底能力

**Non-Goals:**
- 不改变 10 轮下限规则
- 不改变子代理隔离规则

## Decisions

### D1: 摘要格式

```markdown
## 轮间摘要 (Round {N-1} 完成后)

### 已发现问题 (已修复)
- [FP-001] 接口返回值类型不匹配 → 已修复 (Round 2)
- [FP-003] 缺少错误处理 → 已修复 (Round 3)

### 未解决问题
- [FP-005] 边界条件处理待确认 → 升级用户决策

### 覆盖率变化
- traceability.md 编码列: Round 1: 40% → Round {N-1}: 95%

### 本轮建议关注
- FP-005 待用户决策后处理
- 已修复项的回归检查
```

### D2: 注入方式

主 Agent 在每轮子代理返回后提取摘要，注入到下一轮子代理 prompt 的开头。

### D3: 跳过条件

子代理收到摘要后：
- 已修复项：仅做快速回归确认（不重新完整检查）
- 未解决项：重点分析
- 未涉及项：全量检查（兜底）

## Risks / Trade-offs

| 风险 | 缓解 |
|------|------|
| 摘要质量差导致后期轮次遗漏 | 保留全量遍历兜底——摘要仅用于优化注意力分配，不跳过任何工作项 |
| 摘要提取增加主 Agent 负担 | 摘要格式固定化，提取规则简单（从自审报告/产物差异中提取） |
