## 使用说明

此文件用于记录流程指引阶段的错误反馈和 Skills 优化建议。

### 记录时机

- 用户指出指引方向错误
- 用户纠正意图识别结果
- Skill 执行结果与预期不符

### 记录内容

每条记录包含以下字段：
- 发现时间、发现阶段、触发场景
- 用户反馈、改进建议
- 状态（待优化/优化中/已优化）
- 优先级（高/中/低）

### 不主动执行优化

记录完成后不自动触发 Skills 优化流程，需人工评估后执行优化。

## Agent 验收不通过记录

以下记录类型用于 Agent 迭代子代理完成后主 Agent 验收不通过的场景。

### 记录格式

每条验收不通过记录包含以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| 发现时间 | 验收不通过的时间戳 | 2026-05-07 15:30 |
| 发现阶段 | 固定为「Agent 验收」 | Agent 验收 |
| 触发场景 | 验收不通过的具体场景 | 覆盖率不达标 / 产物缺失 / 存在占位符 |
| 覆盖缺口详情 | 缺失的功能点清单或未达标项 | FP-003, FP-007, FP-012 |
| 轮次信息 | 当前轮次 / 最大轮次 | 7/7 (已达上限) |
| 改进建议 | 对下次重跑的调整建议 | 增加接口测试用例覆盖、补充 E2E 场景 |
| 状态 | 待重跑 / 已重跑通过 / 用户放弃 | 待重跑 |
| 优先级 | 高 / 中 / 低 | 高 |

### 验收不通过触发条件

以下条件触发 Agent 验收不通过记录：

1. traceability.md 对应列覆盖率 < 100%
2. 阶段产物文件缺失或格式不正确
3. 产物中存在 TODO/TBD/{待填写} 等占位符
4. Agent 迭代子代理返回但主 Agent 验收不通过
5. 子代理输出 COMPLETED 但验收项未达标

### 处理流程

```
验收不通过:
  │
  ├── 记录 skill-suggestion.md
  │
  ├── AskUserQuestion: "验收不通过，是否重新启动 Agent 迭代子代理？"
  │   ├── 用户选「是」→ 调整 prompt → 重新启动 Agent 子代理
  │   └── 用户选「否」→ 标记 .status.md ⚠️ 需修订 → 阻塞当前阶段
  │
  └── 重跑后验收通过 → 更新记录状态为「已重跑通过」
```
