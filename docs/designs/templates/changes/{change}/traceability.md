## 四、维护规则

1. **创建**: `kflow-design` 阶段基于 functional-designs/index.md 中的 FP 清单初始化空白矩阵
2. **填写**: 各执行阶段完成后，由该阶段的子代理填写对应列
3. **验证**: 阶段门控时读取对应列，检查覆盖率 = 100%
4. **归档**: 归档前最终检查所有列覆盖率 = 100%
5. **各阶段仅写入自身列**，不修改其他阶段已填写的列
6. **E2E 测试列填写时**，SHALL 同步更新 element-coverage-tree.md 中对应状态节点的 `tc_ids` 字段和 `tested` 标记，确保树中 TC-ID 映射与 traceability.md 一致
7. **元素覆盖率验证**：design 阶段门控检查 element-coverage-tree.md 中所有 🎯 状态节点的 TC-ID 覆盖率 = 100%，覆盖率 < 100% 时不得释放门控
