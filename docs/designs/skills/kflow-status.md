# kflow-status（状态总结）

> **版本**: 参见仓库根目录 `VERSION` 文件
> **阶段**: 状态总结（按需阶段）

---

## 基本信息

```yaml
name: kflow-status
description: 任务状态总结 - 汇总所有未归档变更的完成状态。按需阶段，输出格式适配子变更进度矩阵。支持读取 docs/changes/index.md 索引文件。支持子变更5阶段进度展示。
license: MIT
triggers:
  - 状态查看
  - 任务总结
  - 进度汇总
allowed-tools:
  - Bash
  - Read
  - Glob
  - Grep
```

---

## 门控检查

> **机制说明**：门控规则定义在 [core-mechanisms/03-status-and-tasks.md](../core-mechanisms/03-status-and-tasks.md#34-门控规则)

此阶段为独立查询阶段，无前置门控要求。自动扫描 docs/changes/ 目录下的所有未归档变更。支持读取 docs/changes/index.md 索引文件（如存在）。

---

## 输入要求

| 产物 | 图例 | 说明 |
|------|------|------|
| 变更目录 | ✅ 必须 | 自动扫描 `docs/changes/` 目录 |
| docs/changes/index.md | 🔶 条件 | 变更管理索引文件，如存在则提供更丰富的元信息 |

---

## 输出产物

在对话中输出状态汇总表格，不产生文件。

> **模板说明**：输出格式定义在本规格文档中（见下方「输出格式」章节），无需独立的模板文件。
> 所有产物均为会话输出，模板列标记为 N/A（会话输出）。

---

## 执行流程

```
状态总结流程:

┌─────────────────────────────────────────────────────────────┐
│                    STATUS WORKFLOW                           │
├─────────────────────────────────────────────────────────────┤
│  1. SCAN      → 扫描 docs/changes/ 目录                      │
│  2. FILTER    → 过滤未归档变更（排除 archive/ 子目录）        │
│  3. INDEX     → 尝试读取 docs/changes/index.md（如存在）     │
│  │   └── 索引文件存在 → 补充活跃变更列表信息                 │
│  │   └── 索引文件不存在 → 仅从扫描结果获取                   │
│  4. READ      → 读取各变更的 .status.md 文件                 │
│  5. AGGREGATE → 汇总各变更阶段状态和子变更进度                │
│  6. OUTPUT    → 输出状态汇总表格（含子变更进度矩阵）          │
└─────────────────────────────────────────────────────────────┘
```

---

## 索引文件读取

如果 `docs/changes/index.md` 存在，从中获取：
- 活跃变更列表
- 已归档变更列表
- 变更创建时间、类型、影响文件等元信息

索引文件不存在时，从目录扫描结果构建信息。

## 状态值含义与进度计算

| 状态值 | 含义 | 进度计算 |
|--------|------|---------|
| ✅ 完成 | 阶段已完成，产物已生成 | 100% |
| 🔄 进行中 | 阶段正在执行 | 按任务完成比例 |
| ⏳ 待开始 | 阶段未开始 | 0% |
| ⏭️ 跳过 | 阶段被跳过（可选阶段或不适用） | 不计入进度（跳过） |
| ❌ 阻塞 | 阶段被阻塞，无法继续 | 保持当前进度 |
| ⚠️ 有问题 | 阶段完成但有遗留问题 | 按 80% 计算 |
| ⚠️ 需修订 | 阶段产物因需求变更或设计错误需修订 | 按 50% 计算（需回退重做） |

### 整体进度计算规则

```
变更整体进度 = 各阶段进度加权平均

阶段权重:
- 设计探索: 10%
- 原型设计: 0% (跳过) 或 10% (前后端项目)
- 详细设计: 25%
- 计划: 10%
- 编码: 20%
- 代码审查: 5%
- 接口单元测试: 10%
- E2E测试: 10% (前后端) / 0% (纯后端)
- 集成测试: 10%

状态值进度映射:
- ✅ 完成: 100%
- 🔄 进行中: 估算比例
- ⚠️ 需修订: 50%
- ⚠️ 有问题: 80%
- ❌ 阻塞: 保持当前
- ⏳ 待开始: 0%
- ⏭️ 跳过: 移除该阶段权重，重新分配
```
