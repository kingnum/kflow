# KFlow Skills 核心运行机制

> **版本**: 参见仓库根目录 `VERSION` 文件
> **创建时间**: 2026-04-29
> **更新时间**: 2026-06-03

> **加载层级**: 基础层
> **适用阶段**: 全部

本文档定义 KFlow Skills 体系的核心运行机制，包括目录结构、状态文件、任务清单、阶段流转规则、回退机制和条件产物引用规范。

### 3.2 子变更级状态文件格式 (subchanges/{subchange}/.status.md)

```markdown
# 子变更状态：{subchange-name}

## 基本信息
- **所属变更**: {change-name}
- **创建时间**: {YYYY-MM-DD HH:MM}
- **功能点数**: {数量}（≤10）
- **执行类型**: {AFK|HITL}
- **依赖子变更**: {列表或"无"}
- **优先级**: {高|中|低}
- **当前阶段**: {阶段名称}
- **执行轮次**: {N} / 10

## 阶段状态

| 阶段 | 状态 | 开始时间 | 完成时间 | 备注 |
|------|------|----------|----------|------|
| 计划 | ✅ 完成 | 2026-04-29 16:30 | 2026-04-29 17:00 | 任务清单已创建 |
| 编码 | ✅ 完成 | 2026-04-30 09:00 | 2026-04-30 15:00 | TDD 循环完成 |
| 代码审查 | ✅ 完成 | 2026-04-30 15:30 | 2026-04-30 17:00 | kflow-code-review 通过 |
| 接口单元测试 | 🔄 进行中 | 2026-04-30 17:30 | - | 第3/10轮 |
| E2E测试 | ⏳ 待开始 | - | - | (前后端项目) |

> **状态值说明**: ✅ 完成 / 🔄 进行中 / ⏳ 待开始 / ⏭️ 跳过 / ❌ 阻塞 / ⚠️ 需修订 / ⏸️ 等待同步

## 功能点清单

| 序号 | 功能点 | 类型 | 关联功能点 | 实现状态 |
|------|--------|------|-----------|---------|
| 1 | 用户注册 | 后端 | #2 | ✅ 完成 |
| 2 | 用户登录 | 后端 | #1 | 🔄 进行中 |
| 3 | 密码重置 | 后端 | #2 | ⏳ 待开始 |

> **说明**：子变更类型 ≠ 功能点类型。子变更类型（后端子变更/前端子变更）由 design 阶段 DIVIDE 步骤基于包含的 FP 类型自动推断：全部 FP 为「后端」→ 后端子变更；全部 FP 为「前端」→ 前端子变更；混合 → 拒绝划分。FP 类型源自 explore 阶段 functional-designs/index.md 的类型列，两者 SHALL 一致。
```
