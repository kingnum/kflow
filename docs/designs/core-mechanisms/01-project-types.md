本文档定义 KFlow Skills 体系的核心运行机制，包括目录结构、状态文件、任务清单、阶段流转规则、回退机制和条件产物引用规范。

---

## 一、项目类型区分机制

### 1.1 项目类型定义

| 项目类型 | 定义 | 阶段数量 |
|---------|------|---------|
| **前后端项目** | 包含前端工程（存在前端框架依赖、.vue/.tsx/.jsx 文件、前端构建配置等） | 11 阶段（含审计门控） |
| **纯后端项目** | 不包含前端工程 | 9 阶段（含审计门控） |

### 1.2 检测规则

| 检测标识 | 判断依据 |
|---------|---------|
| package.json 前端框架依赖 | react, vue, angular, next, nuxt 等 |
| 前端源文件 | .vue, .tsx, .jsx, .svelte 等文件存在 |
| 前端构建配置 | vite.config.*, webpack.config.*, tsconfig.json 等 |

### 1.3 阶段差异

| 阶段 | 前后端项目 | 纯后端项目 |
|------|-----------|-----------|
| 设计探索 | ✓ 必须 | ✓ 必须 |
| 原型设计 | ✓ 可选（推荐） | ⏭️ 跳过 |
| 详细设计 | ✓ 必须（变更级） | ✓ 必须（变更级） |
| 计划 | ✓ 必须 | ✓ 必须 |
| 编码 | ✓ 必须 | ✓ 必须 |
| 代码审查 | ✓ 必须 | ✓ 必须 |
| 接口单元测试 | ✓ 必须 | ✓ 必须 |
| E2E 测试 | ✓ 必须 | ⏭️ 跳过 |
| 集成测试 | ✓ 必须（变更级） | ✓ 必须（变更级） |
| 审计 | ✓ 必须（归档门控） | ✓ 必须（归档门控） |
| 归档 | ✓ 必须 | ✓ 必须 |
